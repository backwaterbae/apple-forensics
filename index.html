üì• Installation

No dependencies required! The analyzer uses Python 3's standard library.

Step 1: Get the Files

You need these files in the same directory:

  • log_directory_analyzer.py - The analyzer tool
  • macos_security_patterns.txt - 60+ security patterns
  • crash_patterns.txt - 50+ crash patterns
  • malware_patterns.txt - 100+ malware patterns
  • macos_whitelist.txt - 86 legitimate Apple services

Step 2: Make It Executable (Optional)

chmod +x log_directory_analyzer.py
üí° Tip: You can always run it with python3 log_directory_analyzer.py instead of ./log_directory_analyzer.py

üöÄ Basic Usage

Quick Start - Scan Your Logs

python3 log_directory_analyzer.py -p macos_security_patterns.txt

Comprehensive Scan - All Patterns

python3 log_directory_analyzer.py \
  -p macos_security_patterns.txt crash_patterns.txt malware_patterns.txt \
  -v

Scan Specific Directory

python3 log_directory_analyzer.py \
  -d ~/Library/Logs \
  -p macos_security_patterns.txt

Scan Multiple Directories

python3 log_directory_analyzer.py \
  -d ~/Library/Logs /Library/Logs \
  -p macos_security_patterns.txt crash_patterns.txt

System Logs (Requires sudo)

sudo python3 log_directory_analyzer.py \
  -d /var/log \
  -p macos_security_patterns.txt

‚ö° Whitelist Feature (NEW in v1.2)

üéØ Recommended: Always use the whitelist for production scans to filter out false positives from legitimate Apple services like XProtect, Safari, and system components.

Why Use Whitelist?

Security patterns often trigger on legitimate Apple services:

  • XProtect - Apple's malware scanner triggers "malware" patterns
  • imklaunchagent - Input Method Kit triggers "launchagent" patterns
  • Safari components - Browser services trigger persistence patterns

Basic Usage with Whitelist

python3 log_directory_analyzer.py \
  -p macos_security_patterns.txt \
  -w macos_whitelist.txt

Full Scan with Whitelist (Recommended)

python3 log_directory_analyzer.py \
  -p macos_security_patterns.txt crash_patterns.txt malware_patterns.txt \
  -w macos_whitelist.txt \
  -v

What's Filtered:

  • 86 legitimate Apple services
  • XProtect (4 patterns)
  • imklaunchagent (3 patterns)
  • Safari components (4 patterns)
  • System services (40+ patterns)
  • iCloud, Photos, AirPlay, and more

üîß Troubleshooting

‚ùå "No log files found"

# Check directory path
ls -la ~/Library/Logs

# Try absolute path
python3 log_directory_analyzer.py -d /Users/yourname/Library/Logs -p patterns.txt

‚ùå "Permission denied"

# System logs need sudo
sudo python3 log_directory_analyzer.py -d /var/log -p patterns.txt

# Or copy logs first
cp -r /var/log ~/evidence/var_log
python3 log_directory_analyzer.py -d ~/evidence/var_log -p patterns.txt

‚ùå "No patterns loaded"

# Verify pattern file exists
ls -la *.txt

# Check file format (should be UTF-8 text)
file macos_security_patterns.txt

# Use --create-example for valid template
python3 log_directory_analyzer.py --create-example patterns.txt

‚ö†Ô∏è Too many results

# Focus on HIGH severity only
grep "HIGH" log_analysis_*.csv

# Use whitelist to filter false positives
python3 log_directory_analyzer.py -p patterns.txt -w macos_whitelist.txt

# Narrow to specific directory
python3 log_directory_analyzer.py -d ~/Library/Logs/DiagnosticReports -p patterns.txt
